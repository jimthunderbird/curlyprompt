CurlyPrompt Language Tutorial

A curly-brace DSL that compiles to SKILL Markdown — learn every feature with side-by-side examples.

Skill Block

Every .prompt file must contain exactly one skill { } block. All content outside it is ignored. Inside, you define frontmatter (metadata) and a content block.

CurlyPrompt 
skill {
  name: "My Skill"
  description: "A helpful skill"
  content {
    h1:Hello World
    p:Welcome to CurlyPrompt.
  }
}
Markdown Output 
---
name: My Skill
description: A helpful skill
---

# Hello World

Welcome to CurlyPrompt.

Frontmatter Key-Value Pairs

Before the content { block, keys like name, description, license, version, and include are parsed as key: value. Double quotes around values are automatically stripped.

CurlyPrompt 
skill {
  name: "Simple Skill"
  description: "A simple skill for AI Agent"
  license: "MIT"
  version: "1.0"
  content {
    h1:Hello
  }
}
Markdown Output 
---
name: Simple Skill
description: A simple skill for AI Agent
license: MIT
version: 1.0
---

# Hello
Tip: Quotes are optional — both name: "My Skill" and name: My Skill produce the same output.

Header / Body Blocks

As an alternative to flat frontmatter, you can group metadata inside a header { } (or its alias head { }) block, paired with body { } for content.

CurlyPrompt 
skill {
  header {
    name: Simple Header Test
    description: Testing header/body
    version: 2.0
  }
  body {
    h1:Getting Started
    p:follow these steps to begin
  }
}
Markdown Output 
---
name: Simple Header Test
description: Testing header/body
version: 2.0
---

# Getting Started

follow these steps to begin
Tip: head { } is an exact alias for header { }. Use whichever you prefer.

Meta Block

The meta { } block holds arbitrary key-value metadata. It can appear inside header { } or at the flat frontmatter level. It serializes as nested YAML in the output.

CurlyPrompt 
skill {
  name: Multi Meta Test
  description: Test meta attributes
  license: MIT
  meta {
    author: John Doe
    category: utilities
    platform: cross-platform
    priority: high
  }
  content {
    h1:Overview
    p:multi-platform tool
  }
}
Markdown Output 
---
name: Multi Meta Test
description: Test meta attributes
license: MIT
meta:
  author: John Doe
  category: utilities
  platform: cross-platform
  priority: high
---

# Overview

multi-platform tool

Content / Body Block

The content block can be delimited by either content { } or body { }. This is where all markdown elements live.

Tip: When using header { }, pair it with body { }. When using flat frontmatter, use content { }. Both work identically.

Headers (h1 – h6)

h1: through h6: map directly to Markdown # through ######. Each header is followed by a blank line. Inline formatting is applied to header text.

CurlyPrompt 
h1:Main Title
h2:Section Two
h3:Section Three
h4:Section Four
h5:Section Five
h6:Section Six

<!-- with inline formatting -->
h2:very strong:important steps
h3:also strong:important:
Markdown Output 
# Main Title

## Section Two

### Section Three

#### Section Four

##### Section Five

###### Section Six

## very **important** steps

### also **important**:

Header Section Braces

When a header line ends with {, the brace is stripped from the title. The matching closing } is silently consumed. This provides structural grouping only — no effect on Markdown indentation.

CurlyPrompt 
h1:Instruction {
  p:this is a very strong:important instruction
  ul {
    li:step 1
    li:step 2
    li:step 3
  }
}

h2:some more paragraph
Markdown Output 
# Instruction

this is a very **important** instruction

- step 1
- step 2
- step 3

## some more paragraph
Tip: Section braces are purely for readability and editor folding — they don't change the output structure.

Inline Paragraph (p:)

p:TEXT outputs the text followed by a blank line. Inline formatting is applied.

CurlyPrompt 
p:This is a simple paragraph.
p:Use code:npm install to set up.
Markdown Output 
This is a simple paragraph.

Use `npm install` to set up.

Block Paragraph (p { })

Multi-line paragraph block. Each non-empty line inside is emitted with inline formatting. One trailing blank line after the block. Empty lines within are skipped.

CurlyPrompt 
p {
  use italic{emphasis} when highlighting
  and strong{bold} for critical warnings
}
Markdown Output 
use *emphasis* when highlighting
and **bold** for critical warnings

Same-line Paragraph (p{...})

p{text here} extracts content between p{ and } on the same line. Emitted as a paragraph with inline formatting and a trailing blank line.

CurlyPrompt 
p{this is a same-line paragraph}
p{another with strong{important} info}
Markdown Output 
this is a same-line paragraph

another with **important** info

Unordered List (ul { })

Contains li: prefixed lines. Each becomes - Item text. Non-li: lines are ignored. Inline formatting is applied.

CurlyPrompt 
ul {
  li:Docker Engine installed
  li:Docker Compose installed
  li:Access to container registry
}
Markdown Output 
- Docker Engine installed
- Docker Compose installed
- Access to container registry

Ordered List (ol { })

Contains li: prefixed lines. Numbered starting at 1. Inline formatting is applied.

CurlyPrompt 
ol {
  li:strong:Build - Compile source code
  li:strong:Test - Run unit tests
  li:strong:Deploy - Push to production
}
Markdown Output 
1. **Build** - Compile source code
2. **Test** - Run unit tests
3. **Deploy** - Push to production

Standalone li:

li:Text outside a list block becomes - Text. Inline formatting is applied. No trailing blank line.

CurlyPrompt 
li:standalone item outside a list
Markdown Output 
- standalone item outside a list

Code Block (code { })

Lines inside are collected raw. Common leading whitespace is stripped. Brace depth tracking ensures inner {/} don't close the block prematurely. Output uses fenced triple backticks.

CurlyPrompt 
code {
  FROM node:18-alpine
  WORKDIR /app
  COPY package*.json ./
  RUN npm ci
  COPY . .
  CMD ["node", "server.js"]
}
Markdown Output 
```
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
CMD ["node", "server.js"]
```

Brace Depth Tracking

Code blocks can contain { and } characters. The converter tracks brace depth — only a standalone } at depth 0 ends the block.

CurlyPrompt 
code {
  exports.up = function(knex) {
    return knex.schema.createTable('users', (table) => {
      table.increments('id');
      table.string('name');
    });
  };
}
Markdown Output 
```
exports.up = function(knex) {
  return knex.schema.createTable('users', (table) => {
    table.increments('id');
    table.string('name');
  });
};
```

Inline Code (code:)

As a standalone line, code:TEXT wraps text in backticks and appends a blank line. Inline within paragraphs and list items, it uses word-capture to wrap words in backticks.

CurlyPrompt 
<!-- standalone -->
code: npx knex migrate:make name

<!-- inline in paragraph -->
p:Use code:npm install to set up.

<!-- inline in list items -->
ul {
  li:code:main branch is production
  li:Feature branches use code:feature/ prefix
}
Markdown Output 
`npx knex migrate:make name`

Use `npm install` to set up.

- `main` branch is production
- Feature branches use `feature/` prefix

Image — Colon Syntax

img:ALT:PATH becomes ![ALT](PATH). The first : after img: separates the alt text from the path.

CurlyPrompt 
img:Main Dashboard:screenshots/dashboard.png
img:Docker Architecture:docs/docker-arch.png
Markdown Output 
![Main Dashboard](screenshots/dashboard.png)

![Docker Architecture](docs/docker-arch.png)

Image — Brace Syntax

One-line brace form with src and alt key-value pairs. Key order doesn't matter. Values are double-quoted. Spacing between img and { is flexible.

CurlyPrompt 
<!-- comma-separated -->
img{src:"photo.jpg",alt:"a photo"}

<!-- space-separated -->
img{ src:"logo.svg" alt:"company logo"}

<!-- reversed order -->
img{alt:"reverse" src:"reverse.png"}

<!-- flexible spacing -->
img   {src:"spaced.jpg" alt:"space"}
Markdown Output 
![a photo](photo.jpg)

![company logo](logo.svg)

![reverse](reverse.png)

![space](spaced.jpg)

Link — Colon Syntax

link:DISPLAY:URL becomes [DISPLAY](URL). URL must start with http:// or https://.

CurlyPrompt 
p:Visit link:Google:https://www.google.com for searching.
p:See link:rate limit docs:https://api.example.com/docs for details.
Markdown Output 
Visit [Google](https://www.google.com) for searching.

See [rate limit docs](https://api.example.com/docs) for details.

Link — Brace Syntax

One-line brace form with display and url key-value pairs. Key order doesn't matter. Spacing between link and { is flexible.

CurlyPrompt 
p:Visit link{display:"Google" url:"https://www.google.com"} for searching.
p:Go to link  {display:"Docs" url:"https://docs.example.com"} for help.
Markdown Output 
Visit [Google](https://www.google.com) for searching.

Go to [Docs](https://docs.example.com) for help.

strong: (Word-Capture)

strong: captures one or more words and wraps them in **...**. Capture stops before a stop word (like to, for, in, and, or, etc.).

CurlyPrompt 
p:this is a very strong:important instruction
p:Use strong:conventional commits with format
li:strong:Nullable: disabled
Markdown Output 
this is a very **important** instruction

Use **conventional commits** with format

- **Nullable**: disabled
Tip: Word-capture uses [\w.]+ (including dots and hyphens). Stop words include: to, for, in, with, and, or, within, from, by, at, on, of, as, the, is, are, a, an, and many context-specific words.

strong{...} (Inline Brace)

strong{important text} becomes **important text**. Same-line brace form for explicit control over what gets bolded.

CurlyPrompt 
p:strong{this is very important}, rest is ok
p:read the strong{critical section} before proceeding
Markdown Output 
**this is very important**, rest is ok

read the **critical section** before proceeding

italic: (Word-Capture)

italic: works like strong: but wraps in *...*. Same mechanics, different stop words.

CurlyPrompt 
p:This is italic:important and critical
p:Keep migrations italic:small and focused
Markdown Output 
This is *important* and critical

Keep migrations *small* and focused

italic{...} (Inline Brace)

italic{key concept} becomes *key concept*. Same-line brace form.

CurlyPrompt 
p:the italic{key concept} is explained below
h5:Italic italic{styled} h5
Markdown Output 
the *key concept* is explained below

##### Italic *styled* h5

code: (Word-Capture, Inline)

When used inline (not as a standalone element), code: captures words using \S+ (allowing special characters) and wraps in backticks.

CurlyPrompt 
p:Use code:npm install to install packages.
ul {
  li:code:docker build -t myapp . to build the image
  li:code:docker compose up -d to start services
}
Markdown Output 
Use `npm install` to install packages.

- `docker build -t myapp .` to build the image
- `docker compose up -d` to start services

Blockquote

blockquote:TEXT (or its alias bq:TEXT) becomes > TEXT. Inline formatting is applied.

CurlyPrompt 
blockquote:This is a very important quote.
bq:Short alias works the same way.
blockquote:Another with strong:bold content inside.
Markdown Output 
> This is a very important quote.

> Short alias works the same way.

> Another with **bold** content inside.
Tip: bq: is an exact alias for blockquote:. Use whichever you prefer.

Horizontal Rule

A line that is exactly hr outputs --- followed by a blank line.

CurlyPrompt 
h1:Section One
p:Content in section one.
hr
h1:Section Two
p:Content in section two.
Markdown Output 
# Section One

Content in section one.

---

# Section Two

Content in section two.

Table

Contains tr { } row blocks, each containing td: (data) or th: (header) cells. The first row becomes the header, followed by a | --- | ... | separator. Cells support inline formatting.

CurlyPrompt 
table {
  tr {
    td: Name
    td: Age
    td: City
  }
  tr {
    td: Alice
    td: 30
    td: New York
  }
  tr {
    td: Bob
    td: 25
    td: London
  }
}
Markdown Output 
| Name | Age | City |
| --- | --- | --- |
| Alice | 30 | New York |
| Bob | 25 | London |

Include

include: filename.md in the frontmatter adds an include path. Quotes are stripped like all frontmatter values.

CurlyPrompt 
skill {
  name: "Include Test"
  description: "Test include keyword"
  include: global-restriction.md
  content {
    h1:Getting Started
    p:follow the instructions below
  }
}
Markdown Output 
---
name: Include Test
description: Test include keyword
include: global-restriction.md
---

# Getting Started

follow the instructions below

Inline Formatting Application Order

The processFormatting(text) function applies transforms in this exact order:

  1. link{...} — brace syntax links
  2. link: — colon syntax links
  3. strong{...} — brace syntax bold
  4. italic{...} — brace syntax italics
  5. strong: — word-capture bold
  6. italic: — word-capture italics
  7. code: — word-capture inline code

Applied to: headers, paragraphs, list items, and blockquotes.

Tip: Brace syntax is always processed before colon syntax. This means link{...} takes priority over link: within the same text.

Full Example

Here's a complete .prompt file using many features together, and its Markdown output.

CurlyPrompt 
skill {
  name: "REST API Guide"
  description: "API documentation"
  license: "MIT"
  version: "3.1"
  content {
    h1:API Documentation {
      p:This describes the REST API.

      h2:Authentication
      p:Requires a strong:Bearer token.
      blockquote:Tokens expire after 24h.

      h2:Endpoints {
        h3:Get All Users
        p:code:GET /api/users
        ul {
          li:Returns a list of all users
          li:Supports code:page and code:limit parameters
        }

        h3:Create User
        p:code:POST /api/users
        p:Request body:
        code {
          {
            "name": "John Doe",
            "email": "john@example.com"
          }
        }
      }

      h2:Rate Limiting
      p:Limited to strong:100 requests per minute.
      p:See link:docs:https://api.example.com/docs for details.
    }
  }
}
Markdown Output 
---
name: REST API Guide
description: API documentation
license: MIT
version: 3.1
---

# API Documentation

This describes the REST API.

## Authentication

Requires a **Bearer** token.

> Tokens expire after 24h.

## Endpoints

### Get All Users

`GET /api/users`

- Returns a list of all users
- Supports `page` and `limit` parameters

### Create User

`POST /api/users`

Request body:

```
{
  "name": "John Doe",
  "email": "john@example.com"
}
```

## Rate Limiting

Limited to **100** requests per minute.

See [docs](https://api.example.com/docs) for details.
